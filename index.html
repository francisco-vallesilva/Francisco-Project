Predicting Aerodynamic Coefficients from Airfoil Geometry Using Machine Learning

A neural network surrogate for computational fluid dynamics

Francisco Valle-Silva

Mentored by Benjamin Joffe, Georgia Tech Research Institute

Code UIUC Airfoil Database
Abstract

A faster way to ask "how will this wing perform?"

Designing an aircraft wing is an exercise in optimization. Engineers evaluate huge numbers of candidate airfoil shapes across many flight conditions, and the gold standard tool for those evaluations is Computational Fluid Dynamics. CFD is highly accurate, but a single simulation can take hours to days, and a full design study may require thousands of runs. That cost limits how much of the design space anyone can realistically explore.

This project builds a surrogate model: a deep neural network trained to approximate the mapping from airfoil geometry to aerodynamic performance, returning predictions in milliseconds instead of hours. The surrogate does not replace high fidelity simulation. It learns from physics based simulation data, then lets engineers rapidly screen new geometries and reserve expensive CFD runs for the most promising candidates. The same approach is actively used at institutions such as NASA for exactly this reason.

To build the training set, standardized 2D airfoil profiles were run through XFOIL, an established aerodynamic solver, to compute lift and drag coefficients. Each airfoil was rendered as an image, and a ResNet-18 convolutional neural network was trained to predict performance directly from the shape. The result is a complete pipeline from raw geometry to a trained surrogate, built from scratch, that reaches an R² of 0.906 against held out solver data.

Data Pipeline

From coordinate files to training pairs

The pipeline starts with 1573 airfoils from the UIUC Airfoil Database, each stored as a Selig format coordinate file. A plotting program renders every airfoil as a 224 by 224 pixel image, the input size expected by ResNet-18. In parallel, each geometry is run through XFOIL at a fixed flight condition (Re = 1,000,000, Mach 0) to produce a polar: lift and drag coefficients across a sweep of angles of attack.

Not every airfoil survives this process. XFOIL's viscous solver fails to converge on some geometries, and others converge at too few angles to be usable. After matching images to valid labels at an angle of attack of 5 degrees, 854 samples remain. That attrition is a normal part of working with real solver output.

Model

ResNet-18 as an aerodynamic surrogate

The surrogate is a ResNet-18 convolutional neural network with its classification head replaced by a regression head. The network takes a 224 by 224 rendering of an airfoil as input and outputs a predicted lift coefficient. Framing the problem visually, rather than feeding the network raw coordinates, lets the convolutional layers learn spatial features of the geometry, such as camber, thickness distribution, and leading edge shape, that drive aerodynamic behavior.

Training minimized regression error between the network's predictions and the XFOIL labels. The final configuration, selected after thirteen training runs, uses a dropout rate of 0.5 for regularization with a fixed random seed for reproducibility. The full training code is available in the repository linked above.

Results

Predicted lift versus the solver's answer

The scatter plot below is the final model's held out evaluation from training run 13, and it is fully interactive: every faint gold circle is a real evaluation point, and hovering (or tapping) one reads out the values it represents. Each point is one airfoil the network never saw during training. Its position compares the lift coefficient XFOIL computed against the lift coefficient the network predicted from the airfoil's image alone. The dashed line marks perfect agreement.

The training history for the same run shows why the fit holds. Both training and validation error fall steadily and settle without the validation curve turning back upward, so the network is generalizing to unseen airfoils rather than memorizing the training set.

For a screening tool, this is the useful regime: the surrogate ranks candidate geometries reliably enough to narrow a large design space down to a shortlist, which can then be verified with full fidelity simulation.

Experiments

Thirteen runs to the final model

The final configuration was not the first attempt. The baseline, a fully frozen ResNet-18 backbone, reached a validation R² of only 0.37. Unfreezing the last layer group let the network adapt its features to airfoils and jumped performance immediately, and the remaining runs were a systematic sweep: weight decay first, then learning rate, then dropout, watching both validation R² and the gap between training and validation performance.

Two lessons stand out from the sweep. First, run-to-run variance is real: run 2 repeated an earlier configuration and landed at a noticeably different R² purely from random initialization, which is why run 12 fixed the data split with a seed before the final comparison. Second, regularization was the difference between a memorizer and a model: weight decay and dropout together cut the train-validation gap from 0.13 at run 2 to 0.057 by run 12, before the final run traded a little of that gap for the best validation score.

Draw

Draw your own airfoil

Drag the control points to reshape the airfoil, then run the same trained network on your design. Blue points shape the upper surface, orange points the lower surface. Before inference, the outline is rendered as a black 224 by 224 silhouette, the same format as the training images, and fed through the exact model running in the section above.

There is no XFOIL ground truth for a hand-drawn shape, so the number is the surrogate's screening estimate. Shapes far outside the training distribution, such as self-intersecting outlines, will produce unreliable predictions. And a higher CL does not mean a better airfoil: the model predicts lift only. Drag is not considered, so this number says nothing about aerodynamic efficiency, which is the lift-to-drag ratio.
